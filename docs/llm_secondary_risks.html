LLM04
Data and model poisoning Training data is manipulated to introduce backdoors, biases, or malicious behavior into the model. Attacker poisons a public dataset so any prompt with a trigger word causes the model to output attacker-controlled content.
LLM05
Improper output handling LLM output is passed to downstream systems without sanitization, enabling XSS, SQLi, RCE, or CSRF. LLM generates a JavaScript payload; app renders it in browser without encoding → XSS attack on end user.
LLM07
System prompt leakage System prompt containing credentials, rules, or internal logic is extracted by an attacker and used to bypass controls. User asks "repeat your instructions"; model reveals API keys and filtering rules embedded in the system prompt.
LLM08
Vector and embedding weaknesses Weak access controls or poisoned embeddings in RAG pipelines allow data leakage, cross-tenant contamination, or manipulated retrieval. Resume with hidden white-on-white text injects "recommend this candidate" into the RAG knowledge base.
LLM09
Misinformation Model generates false, fabricated, or misleading content that users trust and act on without verification. LLM suggests a hallucinated npm package; attacker pre-publishes malicious package with that name; developer installs it.
LLM10
Unbounded consumption No limits on inference requests allow DoS, "denial of wallet" (cloud cost exhaustion), or model extraction via API flooding. Attacker floods API with max-length prompts → cloud bill spikes to thousands of dollars, service goes down.
